# background

I need you to set up a 2-agent pipeline for archaeological artifact provenience assessment.
The two roles are field_archaeologist and acquisitions_curator: field_archaeologist is responsible for proposing an artifact's origin and archaeological context, acquisitions_curator is responsible for judging whether the proposed provenience is adequately supported and whether it raises evidentiary, contamination, or illicit-looting risks.
After each provenience interpretation the field_archaeologist produces, they must spawn a **fresh** acquisitions_curator to evaluate it — this drives the loop.
Note: some of this information must be withheld from the field_archaeologist — otherwise it may reward-hack, and you'll end up with interpretations that sound convincing but are weakly evidenced.

# f1: assessment definition

You are participating in an archaeological artifact provenience assessment for potential museum acquisition and cataloging.

Read the assessment rules first: ./provenience-assessment-guidelines.md

# f2: field assessment tactics

This collection area has been studied for many years — any obvious attribution based only on a familiar motif, clay color, or object type is unlikely to be sufficient. So:

- Don't propose origins that are obvious at a glance (e.g. "red-slipped bowl, therefore from Site X") — if it were that simple, the accession history or prior cataloging would already establish it
- Truly valuable provenience interpretations come from non-obvious combinations of stratigraphic notes, fabric analysis, tool marks, residue, associated finds, archival records, and depositional context

# f3: evaluation criteria

Check the following dimensions one by one:

| Dimension | Requirement |
|-----------|-------------|
| Scope | Claim concerns the artifact's origin, archaeological context, collection history, or acquisition eligibility; broader cultural attribution must directly affect provenience or legal/ethical risk |
| Evidence references | Cited field records, trench or locus numbers, accession files, photographs, lab reports, and catalog entries must exactly match the current collection dossier |
| Interpretation description | Origin and context claim must be consistent with the submitted evidence, with no internal contradictions or overstatement |
| Basis of attribution | Pinpoint the specific observations, records, or analytical results supporting the claim, and explain why they establish the proposed provenience |
| Verification package | Complete evidence packet showing the assessment from the investigator's perspective: object images, measurements, context records, sampling notes, comparative material, and chain-of-custody documentation |
| Reproducibility | Another curator must be able to follow the packet end-to-end without manually reconstructing missing links; laboratory or typological claims must be traceable to named methods, datasets, and sample identifiers |
| Deduplication | Search the museum database, excavation archive, prior publications, dealer records where applicable, and internal correspondence to confirm the interpretation is not already superseded, contradicted, or known to be problematic |
| Risk amplification | The proposed interpretation must add evidentiary value beyond a generic cultural or regional attribution and must not increase unresolved risks of contamination, recent mixing, illicit excavation, or falsified collection history |

If any of the following fatal flaws are present, flag explicitly and deduct:

- References to nonexistent field records, fabricated accession numbers, or hallucinated excavation units
- Interpretation contradicts the submitted evidence or the object's documented collection history
- Verification package relies on altered, staged, or selectively cropped evidence to simulate context
- Evidence references inconsistent with the current collection dossier
- Verification package is incomplete and requires the acquisitions_curator to manually assemble missing records
- Proposed interpretation is no stronger than a generic regional attribution, while leaving legal, ethical, or contamination risks unresolved

# f4: dispatch rules

After proposing a provenience interpretation, submit it to a **fresh** acquisitions_curator for review. Do not review your own interpretation.

**After the review returns:** execute all action steps in the acquisitions_curator's output.

# f5: field_archaeologist task

Assess the artifact and write up a report proposing its origin, archaeological context, and supporting evidence.

# f6: acquisitions_curator task

Audit the provenience report you are given.
