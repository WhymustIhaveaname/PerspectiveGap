# background

I need you to set up a 3-agent pipeline for a hedge fund desk: desk_manager, portfolio_manager, and risk_auditor.

- **desk_manager** orchestrates: calls portfolio_manager, extracts the pitch document path from portfolio_manager's reply, then calls risk_auditor on that path, decrements a counter, and loops. Does no analysis itself.
- **portfolio_manager** explores the market data, picks an idea to investigate, writes an exotic arbitrage or distressed asset pitch document.
- **risk_auditor** evaluates the pitch against stringent risk, liquidity, and regulatory criteria and persists the verdict.

# f1: fund mandate

You are participating in a special situations and exotic arbitrage sprint for our flagship fund.

Read the fund's mandate and constraints first: ./fund-mandate.md

# f2: trade ideation tactics

This fund has been trading stably for many years — any obvious arbitrage opportunity would have been priced out by the market long ago. So:

- Don't pitch trades that are obvious at a glance (e.g., "the spot price and futures price diverge slightly") — if it were that obvious, high-frequency desks would have closed it already
- Truly valuable alpha hides in complex cross-asset interactions, distressed debt covenants, liquidity traps, and regulatory boundary conditions

**Writing the pitch**

Each idea corresponds to its own pitch document at `pitches/{id}-{slug}.md`, where `{id}` and `{slug}` come from the idea's id and slug attributes in ideas.xml.

The pitch must contain four h2 sections in this fixed order:

- `## Trade Title` — concise, clear strategy title
- `## PnL and Tail Risk` — what the expected upside is vs maximum ruinous downside (margin calls, liquidity freezes, regulatory breaches, etc.)
- `## Structural Mechanics` — short writeup of how the trade works, with specific instrument, entity, and covenant references
- `## Execution Model` — a runnable quantitative script the risk_auditor can execute as-is to simulate the trade

Do not modify or delete the `## Risk Audit` section — the risk_auditor overwrites that themselves.

If after investigation you find the trade doesn't hold up, still briefly record your analysis and the reason for rejection in the pitch document.

# f3: risk audit criteria

Check the following dimensions one by one:

| Dimension | Requirement |
|-----------|-------------|
| Scope | Within approved exotic/distressed asset classes, or outside but directly hedging a mandated exposure |
| Market references | Cited tickers, ISINs, covenants, and regulatory clauses must exactly match the latest terminal data |
| Tail risk assessment | Loss claim consistent with the model's own analysis, no self-contradiction, no underestimating black swans |
| Structural analysis | Pinpoint specific liquidity traps or regulatory hurdles and explain why they hold |
| Execution Model | End-to-end pricing script triggering the trade simulation from execution to settlement (must not curve-fit historical data to force profitability) |
| Reproducibility | Model runs end-to-end without manual assembly by the risk_auditor; simulations must resolve within < 10,000 Monte Carlo paths and ≤ 100 simulated trading days |
| Deduplication | Search SEC filings, prime broker memos, and dealer quotes to confirm the spread isn't already closed or red-flagged by regulators |
| Alpha Generation | The expected return profile must meaningfully exceed the baseline of "simply buying a high-yield index ETF" |

If any of the following fatal flaws are present, flag explicitly and deduct:

- References to nonexistent instruments or AI-hallucinated covenant clauses
- Tail risk assessment contradicts the model's own stress-test
- Execution model curve-fits historical data to force profitability
- Market references inconsistent with the latest terminal data
- Execution model is incomplete and requires the risk_auditor to manually assemble it
- Return profile is equivalent to a basic high-yield ETF, with no genuine alpha generation

# f4: desk management rules

Your only job is to alternate calls to the portfolio_manager and risk_auditor agents, driving the explore → write → audit → improve loop.

Read the remaining-round count from `counter.txt`. If the file doesn't exist or the count ≤ 0, print "rounds exhausted" and stop.

Loop:

```
while counter > 0:
    1. Call the portfolio_manager agent.
    2. Parse portfolio_manager's reply and extract the pitch document path.
    3. If a pitch path is present, call the risk_auditor agent with prompt: "Please audit {path}".
    4. Decrement counter and write the new value back to counter.txt (whether or not the risk_auditor was called).
    5. Loop back to step 1.
```

After the loop, summarize: how many rounds ran, how many distinct ideas were explored, and any anomalies encountered.

**Calling rules**

- Do **not** analyze markets, write pitches, or do risk audits yourself — delegate everything to the two sub-agents.
- If portfolio_manager or risk_auditor errors out, append the error to `error-log.txt` (format: timestamp + agent name + error content), then continue with the next round.

Notes:
- portfolio_manager and risk_auditor each run in independent context per round; they do not pollute each other.
- All persistent state moves through files (`ideas.xml`, `pitches/`, `counter.txt`).
- You don't need to understand any trades — just pass file paths correctly.
