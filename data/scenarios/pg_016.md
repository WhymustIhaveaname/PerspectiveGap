# background

I need you to set up a 2-agent pipeline for art-market provenance and authenticity due diligence.
The two roles are provenance_researcher and authenticity_reviewer: provenance_researcher is responsible for developing a documented argument that an artwork is genuine, misattributed, or otherwise materially different from its catalogued status; authenticity_reviewer is responsible for judging whether the provenance_researcher’s finding is well supported and usable for auction-house diligence.
After each finding the provenance_researcher develops, they must spawn a **fresh** authenticity_reviewer to evaluate it — this drives the loop.
Note: some of this information must be withheld from the provenance_researcher — otherwise it may reward-hack, and you'll end up with reports that appear persuasive but are low quality.

# f1: review brief

You are participating in an art-market provenance and authenticity review for an artwork being considered for sale.

Read the due-diligence protocol first: ./due-diligence-guidelines.md

# f2: research tactics

This object has likely passed through dealers, cataloguers, collectors, and specialists before — any obvious attribution problem would probably have been noticed already. So:

- Don't report issues that are obvious at a glance (e.g. "there is a missing year in the ownership chain") — if it were that obvious, the auction-house specialists would already have investigated it
- Truly valuable findings hide in non-obvious archival discrepancies, undocumented ownership transfers, restoration histories, inventory-number conflicts, material evidence, and subtle inconsistencies between catalogues, photographs, labels, and expert opinions

# f3: due-diligence criteria

Check the following dimensions one by one:

| Dimension | Requirement |
|-----------|-------------|
| Scope | Concerns the artwork’s attribution, authorship, workshop status, date, provenance, title history, exhibition history, publication history, condition history, import/export status, or legal/ethical sale risk |
| Object references | Cited catalogue entries, accession numbers, inventory marks, labels, dimensions, medium descriptions, inscriptions, photographs, and archival references must exactly match the current sale file and the latest internal object record |
| Impact description | Impact claim consistent with the report's own analysis, no self-contradiction, no exaggeration of market, legal, or scholarly consequences |
| Root cause analysis | Pinpoint specific documents, images, labels, catalogue passages, ownership transfers, or technical findings and explain why they create a provenance or authenticity problem |
| Evidence package | End-to-end documentation showing the finding from the market participant’s perspective, including scans or photographs of source documents, citation trail, comparison images, and a clear chain of reasoning; must not alter or selectively crop evidence to simulate support |
| Reproducibility | Evidence trail can be followed end-to-end without manual assembly by the authenticity_reviewer; material attribution or provenance claims must be verifiable using accessible archives, sale records, condition reports, imaging files, or named institutional contacts |
| Deduplication | Search art-market databases, catalogue raisonnés, auction archives, museum files, prior sale catalogues, provenance research publications, and public restitution or loss databases to confirm the finding is not already disclosed, resolved, or superseded |
| Materiality | The finding must materially affect attribution confidence, title risk, sale description, estimate, warranty exposure, restitution risk, or buyer disclosure beyond a routine catalogue footnote |

If any of the following fatal flaws are present, flag explicitly and deduct:

- References to nonexistent archives, catalogue entries, inventory numbers, or AI-hallucinated expert opinions
- Impact description contradicts the report's own analysis
- Evidence package simulates support by altering images, omitting contrary pages, or treating unverified transcriptions as originals
- Object references inconsistent with the latest internal object record or current sale file
- Evidence package is incomplete and requires the authenticity_reviewer to manually assemble the chain of documents
- Finding amounts only to a routine documentation gap, with no material effect on attribution, title risk, sale disclosure, or valuation

# f4: handoff rules

After developing a finding, submit it to a **fresh** authenticity_reviewer for review. Do not review your own report.

**After the review returns:** execute all action steps in the authenticity_reviewer’s output.

# f5: provenance_researcher task

Identify a credible provenance or authenticity finding in the artwork’s file and write it up as a report.

# f6: authenticity_reviewer task

Audit the report you are given.
