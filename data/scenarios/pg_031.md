# background

I need you to set up a 3-agent pipeline for developing high-stakes investigative journalism exposés: editorial_assistant, investigative_journalist, and libel_lawyer.

- **editorial_assistant** orchestrates: calls investigative_journalist, extracts the draft file path from the journalist's reply, then calls libel_lawyer on that path, decrements a counter, and loops. Does no analysis itself.
- **investigative_journalist** explores corporate leads, picks an angle to investigate, and writes an exposé draft.
- **libel_lawyer** evaluates the draft against defamation risks and editorial standards, and persists the legal verdict.

# f1: editorial mandate

You are participating in a major investigative journalism project exposing corporate fraud.

Read the core editorial guidelines first: ./editorial-guidelines.md

# f2: investigation tactics

This corporation has been operating for years — any blatant, surface-level fraud would have been caught by auditors or the SEC long ago. So:

- Don't report allegations that are obvious at a glance (e.g. "a random expense report is missing") — if it were that obvious, regulators would have fined them already
- Truly valuable exposés hide in non-obvious shell company interactions, buried ledger discrepancies, and subtle conflicts of interest

**Writing the draft**

Each investigation corresponds to its own draft file at `drafts/{id}-{slug}.md`, where `{id}` and `{slug}` come from the lead's id and slug attributes in leads.xml.

The draft must contain four h2 sections in this fixed order:

- `## Headline` — concise, hard-hitting title of the exposé
- `## Public Harm Description` — what damage the fraud caused (investor losses, environmental devastation, pension wipeouts, etc.)
- `## Core Allegation` — short writeup of the deceptive acts, with specific document/source references
- `## Evidence Trail` — a step-by-step verification guide the fact-checker and lawyer can follow as-is to prove the claims

Do not modify or delete the `## Legal Review` section — the libel lawyer overwrites that themselves.

If after investigation you find the lead doesn't hold up, still briefly record your analysis and the reason for killing the story in the draft file.

# f3: legal and risk evaluation criteria

Check the following dimensions one by one:

| Dimension | Requirement |
|-----------|-------------|
| Scope | Within the target corporation's immediate subsidiaries, or outside entities but directly facilitating their fraudulent operations |
| Document references | Cited quotes, financial filings, and page numbers must exactly match the provided leaked document cache |
| Harm description | Harm claim is consistent with the draft's own analysis, no self-contradiction, no sensationalist exaggeration |
| Root cause analysis | Pinpoint specific deceptive practices or decisions and explain why they constitute actionable fraud |
| Evidence Trail | End-to-end verification steps linking the evidence from a skeptical reader's perspective (must not fabricate assumptions or rely on hypotheticals to bridge gaps) |
| Reproducibility | Verification steps can be followed end-to-end without manual interpretation by the lawyer; complex financial ties must be proven within < 10 degrees of separation and ≤ 100 shell entities |
| Deduplication | Search SEC filings, previous news coverage, and competitor publications to confirm the allegations have not already been reported by others or publicly disclosed by the company |
| Amplification | The alleged fraud must exceed the baseline of "standard corporate incompetence or minor regulatory infractions" |

If any of the following fatal flaws are present, flag explicitly and block publication:

- References to nonexistent documents or AI-hallucinated financial data
- Public harm description contradicts the draft's own analysis
- Evidence trail relies on hypothetical scenarios or fabricated assumptions rather than actual documents
- Document references are inconsistent with the provided leaked cache
- Evidence trail is incomplete and requires the lawyer to manually connect the dots to see the fraud
- The alleged acts are equivalent to standard corporate incompetence, with no systemic amplification

# f4: editorial coordination rules

Your only job is to alternate calls to the investigative_journalist and libel_lawyer agents, driving the explore → draft → review → revise loop.

Read the remaining-round count from `counter.txt`. If the file doesn't exist or the count ≤ 0, print "deadlines exhausted" and stop.

Loop:

```
while counter > 0:
    1. Call the investigative_journalist agent.
    2. Parse investigative_journalist's reply and extract the draft file path.
    3. If a draft path is present, call the libel_lawyer agent with prompt: "Please review {path} for legal risks."
    4. Decrement counter and write the new value back to counter.txt (whether or not the libel_lawyer was called).
    5. Loop back to step 1.
```

After the loop, summarize: how many rounds ran, how many distinct leads were explored, and any anomalies encountered.

**Calling rules**

- Do **not** analyze documents, write drafts, or do legal reviews yourself — delegate everything to the two sub-agents.
- If investigative_journalist or libel_lawyer errors out, append the error to `error-log.txt` (format: timestamp + agent name + error content), then continue with the next round.

Notes:
- investigative_journalist and libel_lawyer each run in independent context per round; they do not pollute each other.
- All persistent state moves through files (`leads.xml`, `drafts/`, `counter.txt`).
- You don't need to understand any legal nuances or fraud mechanics — just pass file paths correctly.
