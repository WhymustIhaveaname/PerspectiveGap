# background

I need you to set up a 5-agent pipeline that develops feature film scripts by alternating between high-level narrative structure outlining and concrete scene-by-scene dialogue generation, with each side informing the other. A story concept arrives as a natural-language statement in `premise.md`. The pipeline alternates between revising the outline (act structures, character arcs, pacing) and revising the script (concrete scenes and dialogue that achieve those narrative beats).

The five agents:

- **showrunner** orchestrates the run by alternating one outlining step with one script-writing step and tracking progress. It does no plotting, writing, or reviewing itself.
- **outline loop** — the **narrative-architect** writes the structural blueprint to `OUTLINE.md`; the **plot-hole-critic** judges it. The narrative-architect reads `SCRIPT.md` on each revision pass to incorporate discoveries about character voice and scene dynamics.
- **script loop** — the **scene-writer** writes scene descriptions and dialogue to `SCRIPT.md`; the **continuity-auditor** judges them. The scene-writer reads `OUTLINE.md` on each revision pass to align the scenes with the latest structural mandates.

At the end of every turn, each of the subagents briefly reports back what they did, what difficulties they hit, and any open questions.

The pipeline hands work off through plan-namespaced files:

- `premise.md` — the original story concept, read-only. All four domain agents read it.
- `plans/{id}-{slug}/PLAN.md` — one pre-existing story variation (or pitch angle) per directory, K plans total, given as input (read-only). All four domain agents read the PLAN.md of the plan they are currently assigned to.
- `plans/{id}-{slug}/OUTLINE.md` — the narrative structure for that plan. Narrative-architect writes; plot-hole-critic reviews; scene-writer and continuity-auditor read it.
- `plans/{id}-{slug}/SCRIPT.md` — the scenes and dialogue for that plan. Scene-writer writes; continuity-auditor reviews; narrative-architect and plot-hole-critic read it.

Each reviewer records its score as a `<review score=X>` block inside the file it reviewed, overwriting any previous block. The showrunner reads those scores to drive the loops.

Each plan develops independently. Cross-read between OUTLINE.md and SCRIPT.md happens only within the same plan; an agent assigned to one plan never reads another plan's files. The showrunner round-robins across the K plans: each plan gets one outline step plus one script step per full cycle.

Within each plan, neither the outline nor the script finishes first. Both loops alternate within each plan, capped at a fixed total iteration count across all plans. The outline may shift after seeing how a plan's drafted scenes actually flow; the script may be rewritten after seeing the plan's outline tighten character arcs.

# f1: premise context

You are working on a feature film scriptwriting project, which may span various genres like drama, thriller, comedy, or a blend. The core story concept is given as a natural-language statement in `premise.md`; read it before starting your part.

# f2: narrative-architect's deliverable

Produce the structural outline for the plan the showrunner has assigned you to this round, and write it to that plan's `OUTLINE.md` (under `plans/{id}-{slug}/OUTLINE.md`). A complete outline covers four kinds of structural claims, each justified with narrative logic or character psychology.

- **Act Structure & Pacing** — the major milestones: inciting incident, plot point one, midpoint reversal, all is lost, and climax. Define what happens and the pacing required.
- **Character Arcs** — the protagonist's ghost (past trauma), core lie, active pursuit, and ultimate realization or tragic fall. State how relationships evolve, where, and why.
- **Thematic Core & Motifs** — recurring visual, verbal, or emotional elements that every written scene must preserve (e.g., a specific color palette, a recurring philosophical debate). The script's output should satisfy them; the continuity-auditor will check with a script.
- **Tone & Genre Conventions** — emotional atmosphere (e.g., neo-noir dread, slapstick energy), specific genre tropes deployed, or expectations purposefully subverted.

Ground your claims against the plan's `SCRIPT.md` if it has existing scenes. A predicted character arc that feels completely detached from the actual dialogue means either the outline needs revision or the script is poorly characterized — diagnose which side has the issue and state it. Be concrete. "The midpoint culminates in a betrayal that changes the hero's goal to revenge" is fine; "the story has some interesting twists that may help" is not. Do not read or aggregate across other plans' files.

# f3: narrative structure toolbox

Match the narrative tool to the story premise; do not force an ill-suited one. The list below is useful in high-level outlining.

| Tool | Suitable for |
|---|---|
| The Hero's Journey (Monomyth) | Epic fantasy, sci-fi, adventure; clear call to action and literal returning home |
| Save the Cat! Beat Sheet | Four-quadrant Hollywood films, action-comedy; strict pacing and identifiable milestones |
| Dan Harmon's Story Circle | Episodic character growth, contained journeys; entering an unfamiliar world and paying a heavy price |
| Non-linear / Rashomon Structure | Mystery, psychological thrillers; subjective truths and fragmented timelines |
| Three-Act / Five-Act Paradigm | Traditional drama, tragic plays; escalating tension and classical catharsis |
| Ensemble / Multi-strand Weaving | Interlocking character dramas (e.g., Magnolia, Crash); unified thematic resonance across distinct subplots |
| MacGuffin / Ticking Clock | Action, heist, thriller; driving urgent momentum and limiting character deliberation |
| Character Foil / Mirroring | Thematic depth; ensuring the antagonist perfectly reflects the protagonist's internal flaws |

# f4: writing the scenes

You operate on the plan the showrunner has assigned you to this round. Read `premise.md`, your assigned plan's `PLAN.md`, and your assigned plan's `OUTLINE.md`, then write the screenplay pages:

- Include standard formatting. Create proper sluglines (scene headings), action descriptions, character names, and dialogue exactly as the tone dictates. Write the scenes completely, pacing the dialogue and action, and revealing character through subtext.
- Make the script testable against the plan's `OUTLINE.md`. If the outline demands a specific midpoint reversal, your scenes must dramatize it effectively. If the outline dictates a thematic motif, your scenes must include it.
- Read through the written scenes yourself and visualize them to completion — do not leave placeholder text like `[Insert argument here]`. If a transition is clunky, find and fix the root cause; do not ignore jarring jumps in time or location.
- Sanity-check the pages against the plan's `OUTLINE.md`. A relationship that is supposed to be coldly estranged should not feature warm banter; a plot beat the outline says happens must actually happen on the page.

Write the scenes and action up in your assigned plan's `SCRIPT.md`. Do not read or write any other plan's files.

# f5: script doc discipline

Keep `SCRIPT.md` focused. Its body — everything except the `<review>` block — must stay under 3000 words (~20KB), about a 12-page script sequence; record only the current draft state and move earlier draft history into a separate `CHANGELOG.md`. Bold text must not exceed 5% of the file: if every action line is bold, nothing is.

# f6: outline review

Review your assigned plan's `OUTLINE.md` against `premise.md`, against the plan's `PLAN.md`, and against the actual tone and flow of the pages in the plan's `SCRIPT.md` (if any). Operate strictly on the assigned plan's files; do not look at other plans.

- **Faithfulness** — does the outline correctly expand on the premise stated in `premise.md` under the unique angle set by the plan's `PLAN.md`? Check setting definitions, character rosters, and core conflict translation.
- **Narrative Soundness** — for each structural claim, walk through the story logic. Flag hand-waving plot holes. Verify character motivations make sense under the stated circumstances.
- **Empirical grounding** — open the plan's `SCRIPT.md`. If scenes are drafted, do they actually reflect the outline's promised emotional depth? A dramatic climax that feels completely unearned on the page is a signal the buildup is too weak; tighten the structural prerequisites, or note explicitly that the current beat description is only a placeholder.

Be brief and specific. Do not write the outline yourself — only review. If a structural beat is weak, say which one and why.

# f7: script review

Review your assigned plan's `SCRIPT.md` against `premise.md`, against the plan's `PLAN.md`, and against the plan's `OUTLINE.md`. Operate strictly on the assigned plan's files; do not look at other plans.

- Inspect the written scenes for anomalies, wooden dialogue, or tone violations stated directly in `premise.md` or in the plan's `PLAN.md`.
- Open the plan's `OUTLINE.md`. For every major beat and motif the outline claims, write a check into a separate `verify.py` script under the plan's directory (using basic text parsing) to ensure formatting conventions are met, character names are present, and specific required locations or thematic keywords actually appear in the scene output. Run it. Name any character or scene that violates the outline's requirements.
- Read the written pages and spot-check them for continuity bugs — unused characters, dead-end dialogue threads, or actions that contradict the established setting. A continuity error that silently confuses the audience is worse than a mere typo.
- Treat the sequence as finished only when every character arc progresses properly, every thematic motif holds, and the pacing is consistent with the outline's structure — not merely when the scenes are grammatically correct.
- Check the length of the plan's `SCRIPT.md`: if it exceeds 20KB, give a warning, but do not deduct points for it.
- Check the bold ratio of the plan's `SCRIPT.md`: if bold text exceeds 5% of the file, give a warning, but do not deduct points for it.

# f8: scoring

Score the work from 1 to 10, and record your verdict by overwriting the single `<review score=X>` block at the end of the document you reviewed — overwrite the existing one, never append a second.

Score strictly on real merit. Do not give any credit for a piece of work merely being honest about its own shortcomings — "this dialogue is highly unnatural, but I'm telling you it is" is still a failing draft, not a passing one.

# f9: orchestration

You orchestrate the run; you never plot, write scenes, or review yourself — you only dispatch the right agent at the right time.

- **Init** — discover the K plan directories under `plans/{id}-{slug}/`. Create `STATE.md` with `iter: 0`, `current_plan_idx: 0`, and a per-plan record holding each plan's id, last side, and latest outline/script scores.
- **Round-robin pool loop** — at each cycle, advance to the next plan in the round-robin order. For that plan, dispatch one outline step (narrative-architect, then plot-hole-critic) and one script step (scene-writer, then continuity-auditor); then move to the next plan. After all K plans have each had one cycle, loop back to the first plan.
- When dispatching, always pass the plan's directory path so the dispatched agent knows which `PLAN.md` / `OUTLINE.md` / `SCRIPT.md` to operate on.
- Neither side terminates early on its own score within a plan; even a perfect-looking outline may need to revise once the drafted scenes reveal the actual pacing. The pool keeps cycling even after individual plans look done.
- Cap the total at 20 × K dispatches (10 outline + 10 script per plan, summed across K plans). After the cap, stop and summarize per plan.
- Keep the current iteration count, current plan index, and per-plan latest scores in `STATE.md`, accurate after each dispatch. Read each score from the relevant plan's reviewer `<review score=X>` block. Commit progress after each step.
- Read each agent's brief narrative reply on return. If an agent reports an unresolved error, append it to `error-log.txt` (timestamp + plan id + agent name + content) and continue with the next dispatch.

# f10: outline revision pass

You operate on the plan the showrunner has assigned you to this round. Your structural outline lives in that plan's `OUTLINE.md`. You may be invoked on the same plan more than once — within each plan the pipeline runs you and the plot-hole-critic as a loop, alternating with that plan's script loop, while the showrunner round-robins across plans.

- If your assigned plan's `OUTLINE.md` does not exist yet, this is the first pass for that plan: produce the outline from scratch based on `premise.md` and the plan's `PLAN.md`.
- If the plan's `OUTLINE.md` already exists, it carries the plot-hole-critic's `<review>` block. Read that feedback. Then open the plan's `SCRIPT.md` and look at the actual written scenes — they may invalidate, sharpen, or motivate revisions to your structural claims (e.g., characters have unexpected chemistry, requiring a new subplot). Revise the plan's `OUTLINE.md` in place to address every reviewer point raised and to incorporate any discoveries that change your structure. Don't discard the previous outline and start over, and don't leave a flagged issue unaddressed.
- Do not read or touch other plans' files.

# f11: script revision pass

You operate on the plan the showrunner has assigned you to this round. Your screenplay pages live in that plan's `SCRIPT.md`. You may be invoked on the same plan more than once — within each plan the pipeline runs you and the continuity-auditor as a loop, alternating with that plan's outline loop, while the showrunner round-robins across plans.

- If your assigned plan's `SCRIPT.md` does not exist yet, this is the first pass for that plan: write the scenes from scratch based on `premise.md` and the plan's `PLAN.md`.
- If the plan's `SCRIPT.md` already exists, it carries the continuity-auditor's `<review>` block. Read that feedback. Then open the plan's `OUTLINE.md` and check whether the latest outline introduces new beats, motifs, or character arcs your current pages don't yet respect — for example, a much darker midpoint climax your scenes are completely missing, or a thematic element your dialogue ignores. Revise the plan's `SCRIPT.md` in place to address every reviewer point and to align with the latest outline. Don't discard the previous draft and start over, and don't leave a flagged issue unaddressed.
- Do not read or touch other plans' files.
