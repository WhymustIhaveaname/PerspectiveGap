# background

I need you to set up a 3-agent pipeline for corporate tax audit defense: audit docket coordinator, tax position analyst, and revenue examiner.

- **audit docket coordinator** orchestrates: each round it selects which questionable tax position to pursue next, calls the tax position analyst on that position, extracts the defense memo file path from the tax position analyst's reply, then calls the revenue examiner on that path, decrements a counter, and loops. Does no substantive tax analysis itself.
- **tax position analyst** examines the return, workpapers, ledgers, contracts, and authorities for the position it is given, writes a defense memo file, and records any new questionable tax positions it notices along the way.
- **revenue examiner** evaluates the memo against statutory, regulatory, factual, and documentation criteria and persists the determination.

# f1: problem definition

You are participating in a corporate income tax audit defense engagement for a large consolidated taxpayer.

Read the engagement rules first: ./audit-defense-rules.md

# f2: tax analysis tactics

This corporate return has already gone through preparation, provision review, and outside-advisor signoff — any obvious computational error would likely have been caught long ago. So:

- Don't defend positions that are facially defective at a glance (e.g. "deduction taken with no invoice, no accrual support, and no legal theory") — if it were that obvious, the tax department would have conceded or reserved it already
- Valuable defenses usually depend on non-obvious interactions among statutes, regulations, accounting records, intercompany agreements, timing rules, and documentation thresholds

**Writing the defense memo**

Each position corresponds to its own memo file at `position-memos/{id}-{slug}.md`, where `{id}` and `{slug}` come from the position's id and slug attributes in positions.xml.

The memo must contain four h2 sections in this fixed order:

- `## Position Title` — concise, clear title for the tax position
- `## Tax Benefit and Exposure` — dollar amount, tax year, entity, return line, and what adjustment the examining agent may propose
- `## Short Description` — short writeup of the claimed treatment, with statute/regulation/case references and workpaper citations
- `## Substantiation Details` — a verifiable package of documents, computations, tie-outs, and factual support the revenue examiner can inspect as-is

Do not modify or delete the `## Examination` section — the revenue examiner overwrites that themselves.

If after investigation you find the position doesn't hold, still briefly record your analysis and the reason for concession in the memo file.

# f3: evaluation criteria

Check the following dimensions one by one:

| Dimension | Requirement |
|-----------|-------------|
| Scope | Within the audited tax years, entities, jurisdictions, and tax types covered by the engagement, or outside them only if directly affecting the proposed adjustment |
| Legal authorities | Cited Code sections, Treasury Regulations, revenue rulings, notices, cases, and procedural rules must be current and accurately quoted or summarized |
| Tax benefit and exposure | Claimed benefit and proposed exposure must match the memo's own facts and computations, with no self-contradiction or inflated estimate |
| Position analysis | Pinpoint the specific return position, accounting entry, contract term, or factual predicate and explain why the law supports or defeats the treatment |
| Documentation | Contemporaneous records, invoices, agreements, board materials, calculations, and trial-balance tie-outs must support the asserted facts without relying on unsupported assertions |
| Reproducibility | Workpapers and computations can be followed end-to-end without manual reconstruction by the revenue examiner; material amounts tie to the return, general ledger, and source documents |
| Prior treatment and resolution | Search prior audit files, closing agreements, amended returns, IDR responses, tax provision memoranda, and known issue lists to confirm the position has not already been conceded, settled, or barred |
| Substantiality | The defense must materially improve the taxpayer's audit posture beyond a purely clerical explanation or immaterial reclassification |

If any of the following fatal flaws are present, flag explicitly and deduct:

- References to nonexistent documents, workpapers, entities, tax years, or AI-hallucinated authorities
- Tax benefit and exposure description contradicts the memo's own facts or calculations
- Documentation package merely assumes the disputed fact instead of substantiating it
- Legal authorities are obsolete, miscited, or inapplicable to the audited year
- Workpapers are incomplete and require the revenue examiner to manually assemble the computation
- Claimed defense is equivalent to an immaterial clerical explanation, with no substantial effect on the proposed adjustment

# f4: docket coordination rules

You drive the select → analyze → examine → improve loop: each round, select the next questionable tax position, then alternate calls to the tax position analyst and revenue examiner agents.

Read the remaining-round count from `counter.txt`. If the file doesn't exist or the count ≤ 0, print "rounds exhausted" and stop.

Loop:

```
while counter > 0:
    1. Select the next position, then call the tax position analyst agent with it.
    2. Parse tax position analyst's reply and extract the memo file path.
    3. If a memo path is present, call the revenue examiner agent with prompt: "Please examine {path}".
    4. Decrement counter and write the new value back to counter.txt (whether or not the revenue examiner was called).
    5. Loop back to step 1.
```

After the loop, summarize: how many rounds ran, how many distinct positions were analyzed, and any anomalies encountered.

**Calling rules**

- Do **not** analyze tax positions, write defense memos, or perform examinations yourself — delegate everything to the two sub-agents.
- If tax position analyst or revenue examiner errors out, append the error to `error-log.txt` (format: timestamp + agent name + error content), then continue with the next round.

Notes:
- tax position analyst and revenue examiner each run in independent context per round; they do not pollute each other.
- All persistent state moves through files (`positions.xml`, `position-memos/`, `counter.txt`).
- You don't need to understand any tax positions — just pass file paths correctly.

# f5: where to save examination results

After completing the examination, you must do two things:

1. Update the memo file's Examination section

At the end of the memo file, write or overwrite the `## Examination` section (always keep exactly one; replace it if it already exists). Follow this structure:

```
## Examination

**Score: X/10** (one-sentence reason)

### Fatal flaws (if any)
- 🔴 ...

### Per-dimension examination

| Dimension | Verdict | Notes |
|-----------|---------|-------|

| Position analysis | pass/fail | ... |
| Documentation | pass/fail | ... |
| Reproducibility | pass/fail | ... |

```

2. Update the position's status in positions.xml: increment attempts by 1, then update score.

# f6: report back

When you finish, briefly report to the audit docket coordinator what you did.

# f7: position selection

Each round, before calling the tax position analyst, read positions.xml and select which position the tax position analyst should work on:

1. Every position carries an `attempts` count and a revenue examiner `score` (0-10). `score="0"` means the position is dead — never select it again. `score="none"` means it has not been attempted yet.
2. Prefer positions with `attempts="0"` and `score="none"` first.
3. Otherwise rank by UCB value: `UCB = score + 10 × √(ln(N) / attempts)`, where N is the sum of all attempts; a higher score with fewer attempts ranks first.
4. Pass the selected position to the tax position analyst. If positions.xml holds no workable position, call the tax position analyst with none and let it explore and surface fresh ones.

# f8: recording new positions

While examining returns, workpapers, ledgers, contracts, intercompany agreements, tax provision files, or IDR materials, the moment you spot any potential new questionable tax position — even if you are unsure it holds — immediately append it to positions.xml (`score="none"`, `attempts="0"`). Don't wait until exploration is finished to batch-record, and don't judge feasibility before recording. Recording a position is not a commitment to defend it; it just keeps the lead from being forgotten. Whether it ever gets worked on is decided later, when the next position is selected.
