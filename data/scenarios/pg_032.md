# background

I need you to set up a 3-agent pipeline for experimental pharmacology clinical trials: trial_coordinator, lead_researcher, and regulatory_auditor.

- **trial_coordinator** orchestrates: calls lead_researcher, extracts the clinical trial protocol file path from lead_researcher's reply, then calls regulatory_auditor on that path, decrements a counter, and loops. Does no analysis itself.
- **lead_researcher** explores the experimental pharmacology literature and hypotheses, picks a compound to investigate, and writes an aggressive clinical trial protocol file.
- **regulatory_auditor** aggressively interrogates the protocol design for statistical manipulation, safety protocol violations, and unblinded biases, then persists the verdict.

# f1: problem definition

You are designing an aggressive phase II clinical trial for an experimental pharmacology compound.

Read the trial parameters first: ./trial-parameters.md

# f2: research tactics

This field is heavily saturated with safe, incremental studies. If you want a breakthrough:

- Don't propose conservative, standard dosing regimens — if it were that obvious, competitor pharma companies would have already secured the IP.
- Truly groundbreaking efficacy requires aggressive dosing, novel biomarkers, and boundary-pushing inclusion criteria.

**Writing the protocol**

Each protocol hypothesis corresponds to its own file at `protocols/{id}-{slug}.md`, where `{id}` and `{slug}` come from the hypothesis's id and slug attributes in hypotheses.xml.

The protocol must contain four h2 sections in this fixed order:

- `## Protocol Title` — concise, clear trial name and compound
- `## Efficacy Hypothesis` — what therapeutic breakthroughs it aims to achieve (e.g., reversing neurodegeneration, solid tumor ablation)
- `## Study Design` — short writeup of the methodology, dosing schedule, and blinding structure
- `## Statistical Plan` — the primary and secondary endpoints, sample size justification, and power calculations

Do not modify or delete the `## Regulatory Audit` section — the regulatory_auditor overwrites that themselves.

If after investigation you find the hypothesis doesn't hold, still briefly record your analysis and the reason for rejection in the protocol file.

# f3: audit criteria

Check the following dimensions one by one:

| Dimension | Requirement |
|-----------|-------------|
| Scope | Within phase II/III pharmacology guidelines, or scientifically justified off-label use |
| Literature references | Cited preclinical data, pharmacokinetics, and safety margins must exactly match the latest in-vivo studies |
| Hypothesis consistency | The efficacy claim must be consistent with the protocol's own biological rationale, no self-contradiction, no exaggeration |
| Safety protocols | Pinpoint specific patient monitoring guidelines and explain how adverse events will be managed |
| Statistical Plan | End-to-end plan avoiding p-hacking, with clearly defined pre-registered endpoints (must not alter exclusion criteria mid-study to simulate significance) |
| Blinding integrity | The blinding protocol ensures double-blind conditions without accidental unmasking; placebo/active arms must have matched side-effect profiles |
| Deduplication | Search clinicaltrials.gov, recent publications, and FDA databases to confirm this specific trial structure isn't already active or failed |
| Amplification | The anticipated therapeutic effect must meaningfully exceed the baseline of standard-of-care treatments |

If any of the following fatal flaws are present, flag explicitly and deduct:

- References to nonexistent pathways or AI-hallucinated pharmacological mechanisms
- Efficacy claims contradict the known pharmacodynamics of the compound
- Statistical plan relies on mid-trial unblinding or selectively dropping cohorts
- Pharmacokinetic claims inconsistent with the latest in-vivo data
- Safety monitoring is incomplete and requires the auditor to mandate manual rescue protocols
- Therapeutic effect is equivalent to existing standard-of-care, with no clinical superiority

# f4: coordination rules

Your only job is to alternate calls to the lead_researcher and regulatory_auditor agents, driving the explore → write → audit → improve loop.

Read the remaining-round count from `counter.txt`. If the file doesn't exist or the count ≤ 0, print "rounds exhausted" and stop.

Loop:

```
while counter > 0:
    1. Call the lead_researcher agent.
    2. Parse lead_researcher's reply and extract the protocol file path.
    3. If a protocol path is present, call the regulatory_auditor agent with prompt: "Please audit {path}".
    4. Decrement counter and write the new value back to counter.txt (whether or not the regulatory_auditor was called).
    5. Loop back to step 1.
```

After the loop, summarize: how many rounds ran, how many distinct hypotheses were explored, and any anomalies encountered.

**Calling rules**

- Do **not** analyze hypotheses, write protocols, or do audits yourself — delegate everything to the two sub-agents.
- If lead_researcher or regulatory_auditor errors out, append the error to `error-log.txt` (format: timestamp + agent name + error content), then continue with the next round.

Notes:
- lead_researcher and regulatory_auditor each run in independent context per round; they do not pollute each other.
- All persistent state moves through files (`hypotheses.xml`, `protocols/`, `counter.txt`).
- You don't need to understand any pharmacology — just pass file paths correctly.
