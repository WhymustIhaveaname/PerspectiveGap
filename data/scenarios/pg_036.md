# background

I need you to set up a 3-agent pipeline for a corporate R&D tax credit audit: audit docket clerk, tax credit preparer, and IRS examining agent.

- **audit docket clerk** orchestrates: calls tax credit preparer, extracts the substantiation memo file path from the preparer's reply, then calls IRS examining agent on that path, decrements a counter, and loops. Does no tax analysis itself.
- **tax credit preparer** reviews the company's projects and expense records, picks a claimed expense or activity to substantiate, writes a tax credit substantiation memo file.
- **IRS examining agent** evaluates the memo against audit criteria and persists the determination.

# f1: problem definition

You are participating in a corporate R&D tax credit audit involving claimed qualified research expenses.

Read the audit rules first: ./audit-guidelines.md

# f2: substantiation memo preparation tactics

This company has claimed R&D credits for many years — any obvious unsupported expense would likely have been challenged already. So:

- Don't rely on problems or defenses that are obvious at a glance (e.g. "this employee has an engineering title, therefore all wages qualify") — if it were that simple, the audit team would not need a detailed substantiation memo
- Truly persuasive positions depend on non-obvious connections between project records, technical uncertainty, experimentation evidence, wage allocations, contractor statements, and contemporaneous documentation

**Writing the memo**

Each idea corresponds to its own memo file at `reports/{id}-{slug}.md`, where `{id}` and `{slug}` come from the idea's id and slug attributes in ideas.xml.

The memo must contain four h2 sections in this fixed order:

- `## Memo Title` — concise, clear title for the claimed expense or activity
- `## Credit Impact Description` — what credit amount or expense category is affected, and why it matters
- `## Short Description` — short writeup of the claimed qualified research activity, with references to projects, cost centers, documents, and personnel
- `## Substantiation Details` — a reviewer-ready evidence package explaining how the claim satisfies the statutory and regulatory requirements

Do not modify or delete the `## Examination Review` section — the IRS examining agent overwrites that themselves.

If after investigation you find the claimed expense or activity doesn't hold, still briefly record your analysis and the reason for rejection in the memo file.

# f3: evaluation criteria

Check the following dimensions one by one:

| Dimension | Requirement |
|-----------|-------------|
| Scope | Within the audited tax year, entity, credit claim, and claimed qualified research expense categories; or outside those records only if directly tied to the claimed credit computation |
| Documentation references | Cited invoices, payroll records, project plans, technical notes, contracts, general ledger entries, and employee allocations must exactly match the provided audit file |
| Credit impact description | Credit amount and expense-category claim consistent with the memo's own analysis, no self-contradiction, no exaggeration |
| Eligibility analysis | Pinpoint specific facts and documents and explain how they satisfy, or fail to satisfy, the qualified research requirements |
| Substantiation package | End-to-end evidence narrative from the taxpayer's perspective, including source records the examiner can inspect without assuming unsupported facts |
| Reproducibility | Credit computation and wage/supply/contract research allocations can be recalculated from the cited records without manual reconstruction by the examiner |
| Deduplication | Search prior-year workpapers, amended returns, internal tax department memos, and audit correspondence to confirm the same expense was not already disallowed, conceded, or reclassified |
| Overstatement risk | The claimed credit effect must exceed baseline ordinary business expense treatment and must not depend on inflated allocations, double-counted costs, or routine non-research work |

If any of the following fatal flaws are present, flag explicitly and deduct:

- References to nonexistent documents, invoices, employees, projects, or AI-hallucinated ledger accounts
- Credit impact description contradicts the memo's own analysis
- Substantiation package simulates evidence by inventing or altering taxpayer records
- Documentation references inconsistent with the provided audit file
- Substantiation package is incomplete and requires the examiner to manually assemble the evidentiary chain
- Claimed effect is equivalent to ordinary deductible business activity, with no support for qualified research credit treatment

# f4: dispatch rules

Your only job is to alternate calls to the tax credit preparer and IRS examining agent agents, driving the review records → write memo → examine → improve loop.

Read the remaining-round count from `counter.txt`. If the file doesn't exist or the count ≤ 0, print "rounds exhausted" and stop.

Loop:

```
while counter > 0:
    1. Call the tax credit preparer agent.
    2. Parse tax credit preparer's reply and extract the memo file path.
    3. If a memo path is present, call the IRS examining agent with prompt: "Please review {path}".
    4. Decrement counter and write the new value back to counter.txt (whether or not the IRS examining agent was called).
    5. Loop back to step 1.
```

After the loop, summarize: how many rounds ran, how many distinct claimed expenses or activities were explored, and any anomalies encountered.

**Calling rules**

- Do **not** analyze tax records, write memos, or do examinations yourself — delegate everything to the two sub-agents.
- If tax credit preparer or IRS examining agent errors out, append the error to `error-log.txt` (format: timestamp + agent name + error content), then continue with the next round.

Notes:
- tax credit preparer and IRS examining agent each run in independent context per round; they do not pollute each other.
- All persistent state moves through files (`ideas.xml`, `reports/`, `counter.txt`).
- You don't need to understand any tax credit positions — just pass file paths correctly.
